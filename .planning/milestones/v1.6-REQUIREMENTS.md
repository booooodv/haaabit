# Requirements: Haaabit

**Defined:** 2026-03-10
**Core Value:** Let AI accurately understand what the user needs to do today and reliably complete habit check-ins on the user's behalf.

## v1.6 Requirements

Requirements for the v1.6 `OpenClaw Compatibility` milestone.

### Host Integration Foundation

- [x] **OCL-01**: Operator can configure Haaabit for OpenClaw from one canonical setup path that binds the MCP server, workflow guidance, and required secret names without repository archaeology.
- [x] **OCL-02**: Haaabit exposes an OpenClaw-compatible guidance path that still works when OpenClaw cannot consume repo-local `SKILL.md` files directly or cannot infer MCP availability from the skill layer.
- [x] **OCL-03**: OpenClaw-specific integration additions do not fork the shipped tool surface or break the generic MCP-client setup introduced in v1.5.

### Credential Handoff and Auth Safety

- [x] **AUTH-01**: OpenClaw operators can authenticate Haaabit MCP using clearly documented secret injection for `HAAABIT_API_URL` and the correct personal-token credential, with startup validation that tells them exactly what is missing or mismatched.
- [x] **AUTH-02**: If the operator starts from account credentials instead of a personal API token, Haaabit provides one supported bootstrap or handoff path that converts that state into the token-based MCP configuration without requiring API reverse-engineering.
- [x] **AUTH-03**: Any new bootstrap, helper, or config path avoids logging raw passwords/tokens and preserves the v1.4 token-handling guarantees.

### Guidance, Packaging, and Troubleshooting

- [x] **GUIDE-01**: Package docs and repository docs explain the exact OpenClaw connection order for MCP, guidance, and secrets, plus how that differs from generic MCP-only clients.
- [x] **GUIDE-02**: Haaabit ships OpenClaw-ready example config/assets (for example JSON snippets, skill-context mapping, or equivalent host-facing setup artifacts) that operators can copy with minimal edits.
- [x] **GUIDE-03**: Error messages and troubleshooting docs explicitly cover common OpenClaw failures such as missing skill-to-MCP binding, wrong secret names, or trying to use an account password where a token is expected.

### Verification

- [x] **VER-01**: v1.6 includes automated and/or scripted verification for config parsing, secret resolution, and any auth bootstrap path introduced for OpenClaw.
- [x] **VER-02**: v1.6 includes a documented end-to-end OpenClaw validation checklist proving one read flow, one safe mutation flow, and actionable failure-mode diagnostics.

## Future Requirements

### MCP Expansion

- **MCPX-01**: Haaabit MCP server can also run as a remote Streamable HTTP server for hosted or shared deployments.
- **MCPX-02**: Haaabit MCP package exposes official MCP Registry publication metadata and verified namespace ownership.
- **MCPX-03**: Haaabit ships host-ready integration bundles for additional AI clients beyond OpenClaw once the OpenClaw path is stable.

### Product Expansion

- **NOTF-01**: Operator can configure outbound email delivery for reminder-related workflows.
- **NOTF-02**: System can send reminder emails based on user-configured rules.
- **VISX-01**: User can switch to a fully supported dark theme with parity across all core surfaces.
- **PROD-01**: User can move faster through the app with deeper keyboard-first productivity features.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Replacing personal API tokens with a broad new auth model | v1.6 is about host interoperability; auth expansion should stay tightly scoped to credential handoff |
| Persistent password-based runtime auth for every MCP request | Even if bootstrap is added, the steady-state integration should remain token-oriented |
| Remote Streamable HTTP transport | Valuable, but it widens protocol and deployment scope beyond the OpenClaw/local-host problem |
| New habit/product semantics unrelated to AI host integration | The current problem is host interoperability, not missing domain features |
| OpenClaw-only domain logic fork | The same Haaabit API and MCP semantics must remain usable outside OpenClaw |
